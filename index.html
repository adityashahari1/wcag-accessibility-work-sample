Accessibility Testing Checklist

Use this checklist when reviewing pages for WCAG 2.1 AA compliance. Each item maps to one or more success criteria.

  1. Keyboard navigation (SC 2.1.1): Tab through the entire page. Every interactive element must be reachable and operable without a mouse.
  2. Visible focus (SC 2.4.7): A clearly visible focus indicator must appear on every focused element.
  3. Skip link (SC 2.4.1): Press Tab on the page. The skip link appears first and jumps to main content when activated.
  4. Colour contrast (SC 1.4.3): Normal text ≥ 4.5:1, large text ≥ 3:1. Verify with WebAIM Contrast Checker.
  5. Text resize (SC 1.4.4): Zoom to 200%. No content should be cut off or overlap.
  6. Form labels (SC 1.3.1, SC 3.3.2): Each form control must have a programmatically associated label.
  7. Error identification (SC 3.3.1, SC 3.3.3): Submit the form empty. Each error is identified in text, suggests a correction, and is announced by screen readers.
  8. Image alt text (SC 1.1.1): Meaningful images have descriptive alt text. Decorative images have alt="".
  9. Landmark regions (SC 1.3.1, SC 2.4.1): The page contains <header>, <nav>, <main>, and <footer>.
  10. Modal dialog (SC 2.1.2, SC 4.1.2): Opening the modal traps focus. Escape closes the modal and returns focus to the trigger button.
  11. Table structure (SC 1.3.1): The table uses scope="col" headers. Screen readers announce column headers with each cell.
  12. ARIA sort (SC 4.1.2): After sorting, aria-sort on the active column updates to "ascending" or "descending."
  13. Reduced motion (SC 2.3.3 / best practice): Enable "Reduce Motion" in OS settings. Animations stop or use simple fades.
  14. Language (SC 3.1.1): Page has lang="en" on <html>.
  15. Page title (SC 2.4.2): The <title> uniquely describes the page.
